# DevDocs RAG v1.0 Showcase Release

状态：展示版可发布

日期：2026-06-23

## 1. 发布目标

本版本用于 GitHub 展示、面试讲解和本地演示。目标不是声明生产可用，而是让陌生人能在几分钟内理解 DevDocs RAG 的工程价值：

- 面向研发文档的知识库问答，而不是通用聊天壳。
- 文档上传后经过解析、切块、embedding 和状态机入库。
- 回答必须带引用，知识库外问题显式拒答。
- 后台能查看 ingestion、模型调用、检索调试和质量反馈。
- 默认 Mock Provider 可无 key、离线启动；真实 Provider 有单独基线评测。

## 2. 展示版范围

已纳入：

- Spring Boot 3 后端、Vue 3 前端、PostgreSQL 16 + pgvector 三容器 Compose。
- Markdown、TXT、PDF 上传和异步入库。
- pgvector 向量检索、PostgreSQL ILIKE keyword search 与 RRF hybrid search。
- RAG 问答、引用快照、`NO_ANSWER`、`UNGROUNDED` 标记。
- 固定模板项目审查 Review MVP。
- 管理后台：统计概览、ingestion 日志、模型调用日志、回答反馈、检索调试。
- Mock Provider 默认启动，OpenAI-compatible Provider 通过 `.env` 切换。

不纳入：

- 多租户、RBAC、注册、找回密码、刷新 token。
- Kubernetes、Redis、MQ、MinIO、Milvus、Elasticsearch。
- OCR、docx、rerank、SSE 流式、多 Agent、自动写操作。
- 生产级审计、监控、告警和容量规划。

## 3. 冷启动验收

使用展示版配置启动：

```bash
docker compose --env-file .env.example up -d --build
```

本机普通用法仍是：

```bash
cp .env.example .env
docker compose up -d --build
```

## 4. 环境变量审查

`.env.example` 覆盖展示版运行所需配置：

- PostgreSQL：库名、用户、密码、宿主机端口。
- Web 端口：backend、frontend。
- JWT：secret、过期时间。
- 文件存储根目录。
- Chat Provider：provider、base URL、API key、model、temperature。
- Embedding Provider：provider、base URL、API key、model、dimensions、batch size。
- Retrieval：最小相似度、默认 topK、最大 topK。

安全结论：

- `.env` 已在 `.gitignore` 中忽略。
- `.env.example` 不包含真实 API key。
- README 与 demo 文档只展示默认账号和占位配置。
- 真实 Provider key 只能放在本机 `.env`，不得提交。

## 7. 发布前遗留问题

| 问题 | 影响 | 当前处理 |
|---|---|---|
| 正式录屏暂未提供 | GitHub 展示缺少视频入口 | README 明确暂缓，以截图和可复现启动为主 |
| 默认管理员密码固定 | 仅适合本地展示 | README 明确默认账号，生产化不在 v1.0 范围 |
| Mock Provider 不代表模型效果 | 可能误解评测质量 | README 和评测文档明确 Mock 只证明链路 |
| 真实 Provider 阈值仍需更多库外题标定 | 可能多消耗 Chat 调用 | failure cases 已记录，上调阈值列为后续 |
| citation snapshot 元数据不足 | 删除文档后历史引用定位会降级 | failure cases 已记录，后续版本化设计已存在 |

## 8. 展示版验收结论

DevDocs RAG v1.0 showcase 可作为 GitHub 展示版本发布。

- `.env.example` 完整覆盖展示版配置，不泄露真实密钥。
- docs 相对链接无断链。
- 截图、演示脚本、演示语料和评测文档口径一致。
- 新增 `docs/demo-checklist.md` 可用于发布前复核和现场演示。
