# Interview Script

## 30 秒开场

我做的是一个研发知识库与架构审查助手。它不是简单 PDF 问答，而是把项目文档经过解析、切块、embedding、Hybrid Search 和引用校验后，提供两类能力：第一是带来源的 RAG 问答，第二是固定模板项目审查，例如 PRD 与接口文档一致性检查、任务树遗漏风险检查。

技术栈是 Spring Boot 3 单体、PostgreSQL + pgvector、Vue 3 和 Docker Compose。默认 Mock Provider 能离线跑通，真实 Provider 也做过基线评测。

## 3 分钟项目讲法

1. 背景：工程文档分散在 README、架构文档、接口契约和验收记录里，直接问大模型无法保证知识新鲜和可溯源。
2. 入库：用户上传 Markdown/TXT/PDF，后端落盘后走 `PARSE -> CHUNK -> EMBED -> READY` 状态机；失败会写入 ingestion 日志。
3. 检索：问题先 embedding，再在同一 KB 内检索；P1 增加 vector + keyword + RRF，检索调试页能看到 vector、keyword、final score。
4. 问答：top1 低于阈值直接 `NO_ANSWER`，不调模型；进入模型的上下文最多 6 个 chunk；回答必须带 `[n]` 引用。
5. 可观测：后台能看 ingestion 任务、模型调用日志、检索调试和统计卡片。
6. 审查：P2 增加固定模板 Review，不做 Agent；审查结果持久化，包含风险等级、问题、建议和引用来源。

## 5 分钟演示顺序

1. 打开首页，看知识库列表和 Review 入口。
2. 打开知识库详情，展示 READY 文档、chunk 数、检索调试。
3. 进入问答页，问一个库内问题，展示回答状态 `OK` 和引用卡片。
4. 问一个库外问题，展示 `NO_ANSWER`。
5. 打开后台，展示模型调用日志和 ingestion 日志。
6. 打开项目审查页，选择同一 KB，执行“任务树遗漏风险检查”，展示风险等级、发现的问题、建议修改项和引用来源。

## Review 演示问题

PRD 与接口文档一致性检查：

```text
重点关注接口路径、权限字段、状态码和验收口径是否冲突。
```

任务树遗漏风险检查：

```text
重点关注 Phase 计划、验收 Gate、失败路径、截图证据和联调步骤是否遗漏。
```

## 高压追问转场

如果被问“这是不是只是套壳”：

> 我会从三层回答：第一，入库和检索不是黑盒，有自己的 chunk 策略、SQL、Hybrid Search 和 KB 隔离；第二，回答不是只看生成效果，后端做了阈值短路、引用编号校验、`UNGROUNDED` 标记和日志；第三，效果不是只看 Mock，真实 Provider 有单独基线和失败样本复盘。

如果被问“为什么不直接上 Agent”：

> 我现在的 Review 是固定模板，不是 Agent。原因是审查场景首先要可控、可引用、可复现；没有 RAG 质量和日志基础，Agent 只会放大不可解释性。后续即使做 Agent，也应该从只读、人工确认开始。

如果被问“最大短板是什么”：

> 目前最大短板是生产化治理还不完整，包括文档版本化、质量反馈、权限模型、限流和监控告警。这个项目当前定位是简历和答辩里的工程闭环展示，而不是生产 SaaS。
